Overview

This project is a modularized Node.js application utilizing whatsapp-web.js and Express. It acts as both an automated responder (Bot) and an automated sender (REST API).

The system is designed for high stability, featuring pre-flight cleanup checks, daily rotated logging, and automated phone number formatting.

1. Architecture

  • index.js: The core HTTP server. Exposes the endpoints, initializes the WhatsApp Bot, and handles graceful shutdowns.
  • bot.js: The isolated WhatsApp client configuration. Manages Puppeteer arguments for stability, listens to WhatsApp events, and handles automatic bot replies.
  • logger.js: Centralized Winston logger utilizing winston-daily-rotate-file to safely manage daily logs.
  • web/: Contains this documentation, the Swagger API definition, and the Log Viewer interface.

2. REST API Endpoints

You can test all these endpoints interactively via the Swagger UI.

🌟 Universal Target Resolution

In all of the general endpoints below, the number parameter is a universal target. You can provide:

  • An exact Group Name (e.g., "Family Group")
  • An exact Contact Name (e.g., "John Doe")
  • A raw Phone Number (e.g., "08123456789" or "628123456789")

The server automatically resolves the appropriate ID before sending.

Sending a Text Message

Sends a text message to a number, contact, or group.

POST /send-message
{
    "number": "08123456789",
    "message": "Hello from the API!"
}

Sending Media via URL

Fetches media synchronously from a URL and sends it as a WhatsApp attachment.

POST /send-media
{
    "number": "08123456789",
    "file": "https://example.com/image.png",
    "caption": "Check out this image!"
}

Sending Local Media (Upload)

Upload a file directly from your computer/server via multipart/form-data.

POST /send-local-media
Content-Type: multipart/form-data
- number: "08123456789"
- caption: "My uploaded file"
- file: (file attachment)

Group Messaging (Legacy)

Sends messages to a group using the exact Group Name. (Note: /send-message can now also handle groups directly).

POST /send-group-message
{
    "name": "My WhatsApp Group",
    "message": "Announcement message"
}

Clearing Messages

Deletes the chat history on the bot's side.

POST /clear-message
{
    "number": "08123456789"
}

3. Auto-Bot Commands

The system constantly listens to incoming messages. If a match is found, it automatically replies:

  • !ping - Bot replies: "pong"
  • good morning - Bot replies: "selamat pagi"
  • !groups - Bot replies with a formatted list of all groups the bot is currently a member of, including their exact IDs.

4. Logging System

All outgoing messages generated by either the REST API or the Bot are securely logged into daily rotated files located in /logs/YYYY-MM-DD/whatsapp.log. These logs contain metadata like category, recipient, and content.

You can view today's logs directly via the Log Interface.
